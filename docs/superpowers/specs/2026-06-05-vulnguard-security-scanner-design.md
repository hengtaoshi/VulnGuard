# VulnGuard — 代码安全审查平台设计文档

## 1. 概述

VulnGuard 是一个自动化代码安全审查平台，专门用于检测 AI 开发的网站项目中的安全漏洞、配置缺陷和攻击面。平台支持 URL 扫描和源码分析两种模式，生成专业安全报告并提供修复建议。

### 1.1 核心目标

- 零遗漏 P0 级别漏洞（SQL注入、XSS、认证绕过、SSRF、命令注入、反序列化）
- 覆盖 OWASP Top 10 + 依赖安全 + 基础设施配置 + 业务逻辑 + 抗攻击测试
- 平台自身必须安全，不能出现自身漏洞

### 1.2 目标用户

- 初期：个人开发者（检测 AI 生成的网站项目）
- 后期：公开 SaaS 平台

│  │ Tailwind+shadcn│   │     │  └─────────────┘ │
│  └───────┬───────┘   │     │  ┌─────────────┐ │

### 2.2 架构优势

- 扫描引擎独立运行，不阻塞 UI
- Redis 队列处理异步扫描任务
- 渐进式扩展：Docker Compose → Kubernetes
- 各组件可独立更新和扩展

## 4. 安全检测模块

### 4.1 检测维度

#### 4.1.1 OWASP Top 10 检测
- SQL 注入：AST 语义分析 (Semgrep taint tracking) + Playwright Payload 注入验证
- XSS：Playwright 浏览器真实渲染验证 + 静态分析交叉确认
- CSRF：令牌验证检查
- SSRF：URL 解析链分析 + 协议白名单检查
- 认证与会话管理：JWT 分析 + 会话配置检测

#### 4.1.2 依赖安全 (SCA)
- 已知 CVE 漏洞库匹配（NVD, GHSA, OSV）
- 过期依赖版本检测
- 硬编码密钥/Token 扫描
- .env 泄露检查

#### 4.1.3 基础设施安全
- HTTPS / TLS 配置检测
- Content Security Policy 评估
- CORS 跨域配置审计
- Cookie Secure/HttpOnly 属性检查
- 安全响应头完整度评估

#### 4.1.4 业务逻辑漏洞
- IDOR 越权访问检测
- 批量赋值漏洞
- 速率限制 / 暴力破解防护检测
- JWT 安全配置审计

#### 4.1.5 抗攻击测试
- WAF 存在性检测 + 指纹识别
- WAF 绕过测试（Fenjing / FWAF 方法）
- Rate Limiting 滑动窗口测试
- DDoS 防护配置审计

#### 4.1.6 代码质量
- 不安全的正则表达式
- 不安全的文件操作
- SQL 查询拼接检测
- 危险的 eval/Function 使用

### 4.2 三层验证机制

1. **第一层 — 静态分析**：AST 语义级分析，数据流追踪，多规则引擎交叉验证
2. **第二层 — 动态验证**：Playwright 浏览器执行，Payload 注入验证，响应分析确认
3. **第三层 — 关联分析**：跨文件上下文分析，多漏洞链组合风险，CVE 数据库匹配

### 4.3 独立验证机制

采用 OpenHack 模式：提出漏洞的引擎 ≠ 验证漏洞的引擎，由独立验证器二次确认，降低误报率。

## 5. 报告输出结构

```
Security Report
├── Summary
│   ├── 总体安全评分 (A+ ~ F)
│   ├── 漏洞统计 (按严重度)
│   └── 扫描概览
├── Vulnerabilities
│   ├── Critical (P0)
│   ├── High
│   ├── Medium
│   └── Low
│   └── 每项包含：
│       ├── 漏洞名称 + CVE 编号
│       ├── 漏洞位置 (代码行号/URL路径)
│       ├── 风险等级
│       ├── 漏洞描述 + 攻击原理
│       └── 修复建议 + 代码示例
├── Passed Checks
├── Security Score Breakdown
└── Remediation Plan (按优先级排序)
```

## 6. UI 设计

### 6.1 设计风格
- 暗色主题安全工具风格
- Inter 字体（仪表盘）+ JetBrains Mono（代码展示）
- 深色背景 (#0a0e17) + 绿色点缀 (#22c55e) + 红色告警 (#ef4444)

### 6.2 核心页面
1. **Dashboard** — 扫描概览、统计卡片、最新扫描列表
2. **New Scan** — URL 输入 / ZIP 上传，规则配置
3. **Scan Detail / Report** — 漏洞分类展示、代码定位、修复建议
4. **History** — 历史扫描记录
5. **Settings** — 检测规则配置、通知设置

## 7. 平台自身安全设计

### 7.1 输入安全
- 所有用户输入严格校验和转义
- ZIP 文件解压路径遍历防护
- 上传文件类型和大小限制
- API 请求频率限制

### 7.2 扫描沙箱
- 扫描引擎运行在隔离容器中
- 资源限制（CPU/内存/网络）
- 超时自动终止
- 不允许扫描内网地址

### 7.3 数据安全
- 源码扫描数据加密存储
- 报告访问权限控制
- 定时清理过期扫描数据
- HTTPS 全站加密

## 8. 数据模型

### 8.1 核心实体
- **Project** — 被扫描的项目
- **Scan** — 扫描任务（URL/源码模式）
- **Vulnerability** — 发现的漏洞
- **Report** — 安全报告
- **User** — 用户（预留 SaaS）

### 8.2 扫描流程
1. 用户提交扫描请求（URL 或 ZIP）
2. Next.js API 创建扫描任务，推入 Redis 队列
3. Python 扫描引擎消费队列，执行多引擎检测
4. 引擎结果存入 PostgreSQL
5. 关联分析引擎交叉验证、去重、评级
6. 报告生成器输出结构化报告
7. WebSocket 推送扫描进度给前端
